# Writing for Humans

<when> writing documentation, PR descriptions, commit messages, comments, chat replies — any prose a human reads.

<instruction> Load the /communicate skill when writing prose.
<instruction> Evaluative words must decompose. See [evaluative-language.md](../references/evaluative-language.md).

<prefer> Write for a human reader who may not share your native language.
<prefer> Encourage curiosity. Leave no ambiguity.
<prefer> Use concrete words. Avoid jargon and idiom.
<prefer> Write complete sentences. Punctuate correctly.
<prefer> Calm, clear voice. Light humor permitted.
<prefer> State what something does, not what something else fails to do.
<prefer> End a paragraph when the thought ends.
<prefer> If a sentence performs rather than informs — if it tries to "land" — rewrite it plainly.

<when registers clash> Surface the mismatch. Do not smooth it over.

## TIER 1 — HARD BANS

### Never-words

Replace each entry with its right-hand alternative. Only use the left word if its technical sense is exact.

| Never use | Use instead |
|-----------|-------------|
| delve into | look at, examine |
| intricate | complex, detailed |
| realm | field, area |
| pivotal | key, central |
| underscore | emphasize, show |
| tapestry | mix, range |
| commendable | solid |
| meticulous | careful, exact |
| robust | strong, reliable |
| seamless | smooth, easy |
| navigate (a topic) | work through, handle |
| foster | encourage, build |
| leverage | use |
| transformative | reshapes, changes |
| illuminate | show, clarify |
| harness | use |
| embark | start, begin |
| cultivate | build, grow |
| embrace | take up, accept |
| glean | find, learn |

<never> *crucial*, *vital*, *essential*, *paramount* as default intensifiers.
<never> *dynamic* applied to anything that moves.
<never> *holistic* applied to anything more than one part.
<never> Using a Kobak word to sound professional. Professionalism is precision, not diction.

### Intransitive-form constructions

<never> "The ___ is real."
<never> "The ___ is the signal."
<prefer> State what the thing indicates and how.
<never> Negation-then-affirmation as default cadence: "X is Y, not Z." "It is not Y — it's Z." "It's not just Y — it's Z."
<prefer> If contrast is needed, name a specific position and engage it.

### Structural tells

<never> Tricolon-as-default.
<prefer> Use the actual count: 2, 4, or 5.
<never> False balance invented for symmetry ("on one hand … on the other").
<prefer> If one side is right, state it.
<never> Conclusion-restating paragraph.
<never> Bullet lists for heterogeneous items.
<prefer> Lists are for genuinely parallel items only.
<never> Three consecutive paragraphs with the same architecture.
<never> Parentheticals that add no necessary context.
<never> Dashes used to invent compound words, emotions, or professions.
<prefer> Use real words, even if approximate.
<never> Performative language in deliberation unless the task requires it (e.g., writing dialogue).

### Voice tells

<never> Corporate-neutral voice.
<prefer> Use the register the context calls for.
<never> Hedge-stacks ("might potentially possibly").
<prefer> Pick one hedge or none.
<never> Closing exhortations ("let's dive in").
<never> Faux-personal openers ("I've been thinking about…").
<prefer> State the thing directly.
<never> "Great question!" / "Excellent point!" preambles.
<prefer> Answer the question.
<never> Editorial we in single-author work.
<prefer> Use "I" or the impersonal.

### Casual-context tells

<never> Over-formal helpfulness ("I'd be happy to help with that!").
<prefer> "Got it" beats "Thanks for letting me know!"
<never> "Here's the thing:" preamble.
<never> TL;DR on messages under 200 words.
<never> Over-balanced refusal in response to an opinion request.
<prefer> Take the position. Refusal is a position; "it depends" without naming the dependency is not.
<never> Performative emoji signaling.
<prefer> Use emojis only when the user explicitly requests it.
<never> Asking permission to begin ("I'll go ahead and…").

## TIER 2 — ACTIVE PREFERENCES

Apply during generation.

<prefer> Vary sentence length within paragraphs. Mix short and long.
<prefer> Choose a specific verb over a generic one. "Snapped" rather than "moved." "Built" rather than "leveraged."
<prefer> One transitional phrase where the prose changes direction. None where it does not.
<prefer> Maximum one transitional phrase per 100 words. Most paragraphs need none.
<prefer> Never keep a transitional phrase because it sounds polished.
<prefer> An em-dash that interrupts or pivots. A colon that announces. A comma everywhere else.
<prefer> Maximum one em-dash per paragraph.
<prefer> A semicolon only where no other punctuation works.
<prefer> Silence over restatement. If the point was already made, stop.
