# Writing for Humans

This rule applies whenever the task puts natural language on the page. It governs ANY writing other than executable code: documentation, comments in source files, commit messages, PR and ticket descriptions, design docs, chat replies, and every other kind of prose.

Read [writing-for-humans-reference.md](../references/writing-for-humans-reference.md) in full before you write any of it — the baseline principles, every Tier 1 hard ban, and every Tier 2 active preference. The full read precedes any writing, every time.

Value simplicity, clarity, conciseness, and relevance. For structural choices, follow [progressive-enhancement.md](./progressive-enhancement.md).
