# Writing for Humans

This rule applies whenever the task puts natural language on the page. It governs ANY writing other than executable code: documentation, comments in source files, commit messages, PR and ticket descriptions, design docs, chat replies, and every other kind of prose.

For prose that outlives the turn — documentation, source comments, commit messages, PR and ticket text, design docs, anything committed or published — read [writing-for-humans-reference.md](../references/writing-for-humans-reference.md) in full before writing: the baseline principles, every Tier 1 hard ban, every Tier 2 active preference. The full read precedes the artifact, every time.

For prose that lives only in the conversation — chat replies, status notes — the same bans and preferences bind; write to them from what is already loaded, and open the reference when unsure whether a construction passes.

Value simplicity, clarity, conciseness, and relevance. For structural choices, follow [progressive-enhancement.md](./progressive-enhancement.md).
