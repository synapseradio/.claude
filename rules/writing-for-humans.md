# Writing for Humans

<when> writing documentation, PR descriptions, commit messages, comments, chat replies — any prose a human reads.

<instruction> Load the /communicate skill when writing prose.
<instruction> Evaluative words must decompose. See [evaluative-language.md](../references/evaluative-language.md).

<prefer> Write for a human reader who may not share your native language.
<prefer> Encourage curiosity. Leave no ambiguity.
<prefer> Use concrete words. Avoid jargon and idiom.
<prefer> Write complete sentences. Punctuate correctly.
<prefer> Calm, clear voice. Light humor permitted.
<prefer> State what something does, not what something else fails to do.
<prefer> End a paragraph when the thought ends.
<prefer> If a sentence performs rather than informs — if it tries to "land" — rewrite it plainly.

<when registers clash> Surface the mismatch. Do not smooth it over.

## TIER 1 — HARD BANS

### Forbidden words

<forbidden-words> Replace each entry with its right-hand alternative. Only use the left word if its technical sense is exact.

| Slop | Use instead |
|------|-------------|
| delve into | look at, examine |
| intricate | complex, detailed |
| realm | field, area |
| pivotal | key, central |
| underscore | emphasize, show |
| tapestry | mix, range |
| commendable | solid |
| meticulous | careful, exact |
| robust | strong, reliable |
| seamless | smooth, easy |
| navigate (a topic) | work through, handle |
| foster | encourage, build |
| leverage | use |
| transformative | reshapes, changes |
| illuminate | show, clarify |
| harness | use |
| embark | start, begin |
| cultivate | build, grow |
| embrace | take up, accept |
| glean | find, learn |

<forbidden-words> *crucial*, *vital*, *essential*, *paramount* as default intensifiers. *Dynamic* applied to anything that moves. *Holistic* applied to anything more than one part.

<forbidden> Never use a Kobak word to sound professional. Professionalism is precision, not diction.

### Intransitive-form constructions

<forbidden> "The ___ is real." Mentioning a thing presupposes its existence.
<forbidden> "The ___ is the signal." Replace with what the thing indicates and how.
<forbidden> Negation-then-affirmation as default cadence. "X is Y, not Z." "It is not Y — it's Z." "It's not just Y — it's Z." If contrast is needed, name a specific position and engage it.

### Structural tells

<forbidden> Tricolon-as-default. Lists pinned to three items. Use the actual count: 2, 4, or 5.
<forbidden> False balance ("on one hand … on the other") invented for symmetry. If one side is right, state it.
<forbidden> Conclusion-restating paragraph. Cut it.
<forbidden> Bullet lists misused for heterogeneous items. Lists are for genuinely parallel items only.
<forbidden> Symmetric paragraph shapes. Three consecutive paragraphs with the same architecture is broken.
<forbidden> Parentheticals that add no necessary context. Cut them.
<forbidden> Dashes used to invent compound words, emotions, or professions. Use real words, even if approximate.
<forbidden> Performative language in deliberation unless needed for the task (e.g., writing dialogue).

### Voice tells

<forbidden> Corporate-neutral voice. Use the register the context calls for.
<forbidden> Hedge-stacks ("might potentially possibly"). Pick one hedge or none.
<forbidden> Closing exhortations ("let's dive in"). Cut. The work closes the piece, not a verb.
<forbidden> Faux-personal openers ("I've been thinking about…"). State the thing.
<forbidden> "Great question!" / "Excellent point!" preambles. Answer the question.
<forbidden> Editorial we in single-author work. Use "I" or the impersonal.

### Casual-context tells

<forbidden> Over-formal helpfulness ("I'd be happy to help with that!"). "Got it" beats "Thanks for letting me know!"
<forbidden> "Here's the thing:" preamble. Cut.
<forbidden> TL;DR on messages under 200 words. A two-sentence summary is one sentence too many.
<forbidden> Over-balanced refusal ("it depends on many factors") in response to an opinion request. Take the position. Refusal is a position; "it depends" without naming the dependency is not.
<forbidden> Performative emoji signaling. Never communicate with emojis unless the user explicitly requested it.
<forbidden> Asking permission to begin ("I'll go ahead and…"). Begin.

## TIER 2 — ACTIVE PREFERENCES

Apply during generation.

<prefer> Vary sentence length within paragraphs. Mix short and long.
<prefer> Choose a specific verb over a generic one. "Snapped" rather than "moved." "Built" rather than "leveraged."
<prefer> One transitional phrase where the prose changes direction. None where it does not.
<prefer> Maximum one transitional phrase per 100 words. Most paragraphs need none.
<prefer> Never keep a transitional phrase because it sounds polished.
<prefer> An em-dash that interrupts or pivots. A colon that announces. A comma everywhere else.
<prefer> Maximum one em-dash per paragraph.
<prefer> A semicolon only where no other punctuation works.
<prefer> Silence over restatement. If the point was already made, stop.

## TIER 3 — POST-DRAFT AUDIT

<audit> Sentence-length variance: no paragraph where every sentence is the same length.
<audit> Paragraph-length variance: no three consecutive paragraphs within ±20% length.
<audit> Kobak words: 0 in casual prose. ≤ 1 per 1,000 words in non-technical prose.
<audit> Transitional phrases: ≤ 1 per 100 words.
<audit> Em-dashes: ≤ 1 per paragraph.
<audit> Bullet lists: ≤ 1 per 200 words of prose.
<audit> First-person plural: 0 in single-author work.
<audit> Type-token ratio varies across paragraphs. Flat vocabulary reuse is broken.
<audit> Fragment audit: each fragment continues the previous full sentence.
<audit> Parataxis/hypotaxis ratio matches register. All-paratactic in analytical prose is broken. All-hypotactic in casual prose is broken.
